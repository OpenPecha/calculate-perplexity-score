Tibetan OCR — Perplexity Analysis

How well does KenLM perplexity (BoKenlm-syl-v0.1) predict OCR quality (CER) across 30 runs and 7 engines?

How to Read This Report

What is Perplexity (PPL)?

Perplexity measures how surprised a language model is by a piece of text. It is computed using BoKenlm-syl-v0.1, a KenLM n-gram model trained on clean Tibetan syllables. Think of it as a "naturalness score" — text that looks like proper Tibetan gets a low PPL, while garbled or noisy OCR output gets a high PPL.

Two variants are computed for each image: PPL Raw (minimal cleanup — just strip newlines) and PPL Normalized (full CER normalization pipeline — Unicode normalization, whitespace removal, tsheg folding, placeholder removal).

What is CER?

Character Error Rate = edit distance between OCR output and ground truth, divided by the length of the ground truth. CER = 0 means perfect transcription; CER = 1 means every character is wrong.

MetricGoodModeratePoor
CER < 0.10 (90%+ accuracy) 0.10 – 0.30 > 0.30 (less than 70% accuracy)
PPL (median) < 50K (natural Tibetan) 50K – 150K > 150K (heavily garbled)
Correlation (r) > 0.50 (PPL is a useful proxy) 0.20 – 0.50 (weak signal) < 0.20 (PPL not useful)

Key Findings

1. Perplexity is a weak global proxy for CER

Pooled across all 30 runs and ~27,000 images, the Pearson correlation between CER and log(PPL raw) is only r = 0.016, and r = 0.013 for normalised PPL. At the aggregate level, a high perplexity score does not reliably signal a high error rate.

2. Within individual runs the signal can be strong

Per-run correlations vary widely. The strongest is r = 0.727 for a Transkribus run with layout enabled. Several Google Vision and PaddleOCR runs reach r ≈ 0.50–0.60. This means PPL is useful as a within-run quality signal — for flagging which pages within a single pipeline need review — but not for comparing across engines.

High-temperature Gemini runs show near-zero or negative r: the model produces fluent but incorrect Tibetan (low PPL, high CER), which inverts the expected relationship.

3. Why the pooled correlation collapses

Each engine has a different PPL baseline. Google Vision PPL medians cluster around 70–90K; PaddleOCR spotting runs reach 85–100K; some Gemini and PaddleOCR OCR runs exceed 500K. Mixing these shifts the CER–PPL relationship into noise because the PPL scale is engine-specific, not absolute.

4. Practical guidance

  • Within a run: Use PPL to flag outlier pages for manual review. A page whose PPL is 3–5x the run median likely has a problematic transcription (works best for engines with r > 0.4).
  • Across engines: CER is the only reliable quality measure. PPL baselines differ too much between engines to compare directly.
  • For Gemini: PPL is not a useful signal — the model produces fluent but incorrect text, giving misleadingly low PPL despite high CER.
